git config --global user.name blabla - прописуємо ім'я яке буде відображатися в колонці автора git config --global user.email blabal@bla - пошта яка відображається в інформації про виконаних вами комітів. git init - ініціалізація гіта в данній папці git status - перегляд змін і створення коммітів git add - вказуєм файл який хочемо закомітити. git commit - створюємо зліпок файла (зберігаємо зміни). git log - перегляд історії зміни файла. git checkout - переключити на вибрану гілку і обновити робочу теку до її стану. git branch - видалити дану гілку. git push - завантажуєм всі зміни локальної гілки в віддалений репрозиторій. git pull - завантажуєм історію із віддаленого репрозиторія і обєднуємо її з локальною. git merge - вносимо зміни вказаної гілки в дану гілку. git clone - завантажити репрозиторій разом із його історією змін. gitignore - вказуємо що буде ігнорувати гіт. рядки чи файли. ігнорує - не слідкує за змінами. git revert - відміна нового коміта якщо він не потрібен git reset - відміняє всі зміни в робочій папці git rebase - працює як merge, але не зберігає історію комітів. git cherry-pick - перетягнути коміт з іншої гілки в свою. git stash - якщо нам не потрібно робит коміт, але потрібно зберегти поточну версію змін.
